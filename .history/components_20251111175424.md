Follow the instruction whenever you create a component

## 1. TypeScript Foundation

- Extend native props: `React.ComponentProps<"element">`

## 2. ClassName Management

- Always use `cn()` utility (twMerge + clsx)
- Accept `className` prop for extensibility

## 3. Variant System (CVA)

- Use Class Variance Authority for variants
- Always provide `defaultVariants`

## 4. Props Pattern

- Destructure known props, spread rest
- Accept variant props via `VariantProps<typeof variants>`
- Always forward props with `{...props}`

## 5. Polymorphic Components

- Use `Slot` from `@radix-ui/react-slot`
- Provide `asChild` prop for flexibility

## 6. Data Attributes

- Add `data-slot` to identify parts
- Use descriptive names (e.g., "button", "dialog-content")

## 7. Accessibility First

- Use semantic HTML
- Include proper ARIA attributes
- Support keyboard navigation
- Add screen reader text with `sr-only`

## 8. Focus States

- Use `focus-visible` not `focus`
- Apply: `focus-visible:ring-[3px] focus-visible:border-ring focus-visible:ring-ring/50`
- Handle invalid: `aria-invalid:ring-destructive/20 aria-invalid:border-destructive`

## 9. State Management

- Support disabled: `disabled:pointer-events-none disabled:opacity-50`
- Handle invalid with `aria-invalid`
- Use data attributes: `data-[state=open]`

## 12. Composition Pattern

- Break into composable parts
- Export all sub-components
- Consistent naming (Card, CardHeader, CardContent)

## 13. Context for Complex Components

- Use React Context for shared state
- Create custom hooks (e.g., `useFormField`)
- Throw errors if context missing

## 14. Responsive Design

- Mobile-first approach
- Use: `md:text-sm`, `sm:max-w-lg`
- Container queries where appropriate

## 17. Versioned Imports

- Include version numbers
- Example: `@radix-ui/react-dialog@1.1.6`

## 18. Export Pattern

- Named exports only (no default)
- Export variants for external use
- Export types when needed
