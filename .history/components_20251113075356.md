# üß† 2. TypeScript Rules

- Always extend native HTML element props:


# üé® 3. ClassName & Styling Rules

- Every component MUST accept `className`.
- MUST merge classes using `cn()`.
- NEVER hardcode:
  - colors
  - Typography
- MUST use custom design token via Tailwind classes, if not there use hex arbitary value


# üß¨ 4. CVA Variant System (REQUIRED)

All visual variants must use **Class Variance Authority (CVA)**.

Rules:
- Create a `componentVariants` using `cva()`
- Must define `defaultVariants`
- Must export variant function for external usage
- Accept variant props via:

```ts
VariantProps<typeof componentVariants>
```

Common variants:
- `variant: default | outline | subtle | ghost`
- `size: sm | md | lg`
- `tone: destructive | success | neutral`

# üîÑ 5. Polymorphic Components (`asChild`)

Every interactive component MUST support:

```ts
asChild?: boolean
```

Use:

```tsx
import { Slot } from "@radix-ui/react-slot";
const Comp = asChild ? Slot : "button";
```


# üèóÔ∏è 6. Stateless UI Components (NO LOGIC)

The component file `<Component>.tsx` MUST be purely presentational.

Rules:
- NO `useState`, `useEffect`, or logic inside
- Only render UI / DOM
- Forward all props (`{...props}`)
- Merge `className`
- Support all ARIA attributes

Example:

```tsx
export function Button({ className, variant, size, ...props }) {
  return (
    <button
      className={cn(buttonVariants({ variant, size }), className)}
      {...props}
    />
  );
}
```

---

# üß† 7. Hooks = Behavior Layer (Logic Lives Here)

All interactive or behavioral logic MUST be extracted into hooks.

Example hooks:
- `useDisclosure`
- `useDropdown`
- `useKeyboardNav`
- `useOutsideClick`
- `useFocusTrap`
- `useControllableState`

Hooks MUST:
- Return `{ state, actions }`
- Be reusable
- Contain no JSX

---

# üß© 8. Composition Pattern

Large components MUST be built from composable subcomponents.

Example:

```
Card
  CardHeader
  CardTitle
  CardContent
  CardFooter
```

Rules:
- Each subcomponent is its own function
- Use `data-slot` attribute on each subcomponent
- MUST export all subcomponents

Example:

```tsx
<div data-slot="card-header" />
```

---

# üß¨ 9. Data Attributes (Required)

Components MUST use descriptive data attributes for styling, theming, and state.

Examples:

| Purpose | Attribute |
|---------|-----------|
| slot | `data-slot="button"` |
| variant | `data-variant="outline"` |
| size | `data-size="sm"` |
| state | `data-state="open"` |
| disabled | `data-disabled` |

# ‚ôø 10. Accessibility Rules (MANDATORY)

Every component MUST be accessible by default.

### Required:
- Semantic HTML first
- Support all ARIA props
- Add `aria-label` for icon-only buttons
- Keyboard navigation must Just Work‚Ñ¢
- MUST use visible focus ring:

```
focus-visible:ring-[3px]
focus-visible:ring-ring/50
focus-visible:border-ring
```

### Required Keyboard Support:
- Enter
- Escape
- Space
- ArrowUp / ArrowDown
- Tab / Shift+Tab

---

# üö® 11. Error & Validation States

Input-like components MUST support:

```
aria-invalid="true"
aria-invalid:border-destructive
aria-invalid:ring-destructive/20
```

---

# üß± 12. Disabled State Rules

Disabled components MUST use:

```
disabled:pointer-events-none
disabled:opacity-50
```

Disabled = no hover, no focus, no keyboard.

---

# üì¶ 13. Export Rules

- Named exports ONLY

Example:

```ts

export * from "./Button.types";
export * from "./Button.hooks";
```
